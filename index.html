Why Logic Matters for AI Agents

A Logical Agent is different from a simple reflex agent. It maintains a Knowledge Base (KB) and uses inference to derive new facts.

The KB Agent Cycle:
1. Agent PERCEIVES environment → "It is raining"
2. TELL(KB, "Raining") → KB stores the fact
3. ASK(KB, "What action?") → Inference Engine reasons
4. KB knows: Raining ⇒ TakeUmbrella → derives TakeUmbrella
5. Agent PERFORMS action → Takes umbrella
6. TELL(KB, "Did(TakeUmbrella)") → KB updated

Propositional Logic vs First-Order Logic

Understand the fundamental difference: PL deals with simple true/false facts, while FOL can express objects, relationships, and general rules.

Key Equivalences (Must Memorize for Exam!)

P ⇒ Q ≡ ¬P ∨ Q ← MOST IMPORTANT! (Implication Elimination)
P ⇔ Q ≡ (P⇒Q) ∧ (Q⇒P) ← Biconditional Elimination
¬(P∧Q) ≡ ¬P ∨ ¬Q ← De Morgan's Law 1
¬(P∨Q) ≡ ¬P ∧ ¬Q ← De Morgan's Law 2
¬¬P ≡ P ← Double Negation
P ⇒ Q ≡ ¬Q ⇒ ¬P ← Contrapositive

Inference Rules — Interactive Demos

Modus Ponens
A ⇒ B, A ∴ B
If A implies B and A is true, then B is true
Modus Tollens
A ⇒ B, ¬B ∴ ¬A
If A implies B and B is false, then A is false
And-Introduction
A, B ∴ A ∧ B
If A is true and B is true, then A AND B is true
And-Elimination
A ∧ B ∴ A (or B)
If A AND B is true, then A is true individually
Or-Introduction
A ∴ A ∨ B
If A is true, then A OR B is true (for any B)
Resolution
A ∨ B, ¬A ∨ C ∴ B ∨ C
Cancel complementary literals, combine the rest

First-Order Logic — Construction & Operators

FOL adds objects, quantifiers, predicates, and functions to propositional logic. Here are the building blocks:

FOL Components

ComponentSymbolExample
Constantsa, b, JohnJohn, 2, Pune

Quantifiers

QuantifierSymbolRead as
Universal∀"For all" (paired with ⇒)
Existential∃"There exists" (paired with ∧)
Pairing Rule (EXAM CRITICAL!):
∀ always pairs with ⇒ (implication)
∃ always pairs with ∧ (conjunction)
Mixing these up is the #1 student mistake!

Resolution Proof Engine

The Resolution algorithm proves statements by contradiction: assume the negation, convert to CNF, resolve until empty clause.

Steps:
1. Given: KB (knowledge base) and goal α to prove
2. Negate the goal: Add ¬α to the clauses
3. Convert to CNF: Replace ⇒ with ¬P ∨ Q, distribute ∨ over ∧
4. Resolve: Find complementary literals (P and ¬P), cancel them
5. If EMPTY CLAUSE {} is derived → CONTRADICTION → α is PROVED!
